## Git & GitHub

1. Git branch
    - `main` : 기본 브랜치
    - `github/` : 코드 외 수정 브랜치
    - `feat/` : 기능 개발 진행용 브랜치
    - `bugfix/` : 버그 수정용 브랜치
    - `test/` : 테스트용 브랜치
2. Commit / Pull request
    - `Feat` : 새로운 기능 추가
    - `Fix` : 버그 수정
    - `Design` : 디자인 및 UI 수정
    - `Docs` : 문서 (README, 메뉴얼 등)
    - `Test` : 테스트 코드
    - `Refactor` : 코드 리팩토링 (기능 변화 없이 성능 개선)
    - `Style` : 코드 의미에 영향을 주지 않는 변경 사항
    - `Chore` : 빌드, 설정 파일
    - `Comment` : 주석 추가
